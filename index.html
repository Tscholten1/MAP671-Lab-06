    This project moved into a new cartographic area that involved the process georeferencing images (satellite, photos, basic pictures) that capture an area or portion of a “real-world” geographic area. A second cartographic technique involved creating new vector data using the Qgis suite of tool found in the Layer>Create Layer tab. An aerial photograph provided by a balloon camera on the campus of the University of Kentucky was georeferenced to the specific campus area and a raster GIS layer was generated that provided a source layer from which to digitize new vector data.

    Key lessons learned:

  • Applying a transformation to a static image
  • Creating control points of known locations on the static image
  • Using a Python script (provided in class material) to bulk load a variety of basemaps and reviewing these in the XYZ Tiles feature of Qgis
  • Using Georeferencer GDAL that allows manipulation of images and creating raster layers.
  • Working with unreferenced image with no coordinate information and using the GDAL Georeferencer (located in the Raster tab) to align it with a coordinate system
  • Adding multiple ground control points using Add Point feature (found in the Georefencer tool)
  • Creating vector data layers such as points, lines, and polygons using Qgis tools (Layer>Create Layer)
  • Effort was also spent creating the proper visual hierarchy for the maps and ensuring an attractive map product. Coordinate reference system used is North America Equal Area (EPSG: 102008). Basemap: Raisz Landform map provided by MAP671 instructor. Other data provided by The Appalachian Trail Conservancy (trail shapefile), Natural Eartch (10m graticule).
